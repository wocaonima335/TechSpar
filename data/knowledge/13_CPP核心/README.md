# C++ 核心

> 第 1 版知识库入口 | 面向系统编程、客户端、音视频、基础设施方向的 C++ 面试训练

## 主题定位

C++ 核心强调语言层能力，不和 STL 题库重复。这里重点覆盖对象模型、资源管理、模板机制和现代 C++ 常见设计思路，目标是让专项训练能围绕“原理 + 代码语义 + 工程后果”出题。

## 1. 语法基础与对象模型

- 基本类型、作用域、声明与定义分离，要求能解释编译期名称查找和链接期符号解析的区别。
- 类、结构体、构造函数、析构函数的语义，重点理解对象什么时候被创建、拷贝、移动和销毁。
- 左值、右值、左值引用、右值引用是现代 C++ 的基础语义，直接影响重载决议和资源转移。
- `const`、`constexpr`、`consteval` 的使用边界需要分清，面试常追问它们分别发生在编译期还是运行期。

## 2. 指针、引用与生命周期

- 指针保存地址，可为空、可重绑定；引用更像别名，一旦绑定通常不再改变。
- 野指针、悬空指针、重复释放的根因都和生命周期管理失控有关，面试会结合函数返回值和容器扩容追问。
- `this` 指针、本对象地址、成员访问语义要能讲清，尤其是成员函数背后的调用形式。
- 数组指针、函数指针、指针与数组退化规则是 C/C++ 交界处的高频考点。

## 3. 内存管理与 RAII

- 栈区、堆区、静态存储区、常量区的典型特征要能用“分配时机、释放责任、访问成本”来区分。
- `new/delete` 和 `malloc/free` 的差异不只是语法，而是是否调用构造析构、类型系统是否参与管理。
- RAII 的核心是“资源绑定对象生命周期”，资源可以是内存、锁、文件句柄、socket，而不只是堆内存。
- 异常安全要和 RAII 一起理解，常见追问是为什么析构函数不能随意抛异常。

## 4. 智能指针与资源所有权

- `unique_ptr` 表达独占所有权，适合清晰建模资源归属，也是默认首选智能指针。
- `shared_ptr` 通过引用计数共享所有权，但循环引用会导致对象无法释放，需要 `weak_ptr` 打破环。
- `make_unique`、`make_shared` 的价值在于减少裸指针暴露并提升异常安全。
- 面试常用场景题考查“什么时候不用智能指针”，例如对象不拥有资源、对象生命周期由上层统一控制时。

## 5. 面向对象与多态

- 封装、继承、多态是基础概念，但更重要的是它们在二进制布局和接口设计上的代价。
- 虚函数通过虚表实现动态绑定，常被追问虚表指针大致何时写入对象。
- 纯虚函数、抽象类、虚析构函数要结合“通过基类指针删除派生类对象”这个经典场景理解。
- 重载、重写、隐藏是三个不同概念，面试里经常故意混用术语。

## 6. 模板与泛型编程

- 函数模板、类模板、模板特化、偏特化体现的是“编译期生成代码”的泛型能力。
- 模板实例化通常发生在编译阶段，因此错误信息长、排查难，这是 C++ 模板的真实工程成本。
- `typename`、`template` 关键字在依赖名上下文中的作用属于高频细节题。
- 泛型的目标不是炫技，而是在零额外运行时开销的前提下复用算法和数据结构。

## 7. 移动语义与现代 C++

- 移动语义的本质是把“可转移资源”从一个即将结束生命周期的对象搬到另一个对象中。
- 拷贝构造、移动构造、拷贝赋值、移动赋值和析构函数共同构成资源类的语义边界，常见总结是三法则和五法则。
- `std::move` 本身不移动，只是把表达式显式转换为右值，真正是否发生移动取决于类型实现。
- 完美转发、万能引用和引用折叠是模板代码中保留值类别的关键机制。

## 8. 工程实践与常见陷阱

- 头文件和源文件分离、前向声明、包含保护是减少编译依赖和符号冲突的基本习惯。
- ODR（One Definition Rule）违规会带来链接错误或未定义行为，是大型工程常见坑点。
- 宏、内联函数、命名空间要结合“可维护性”和“二进制兼容性”一起考虑。
- 调试 C++ 问题时，内存越界、未定义行为、对象切片、悬空引用通常比语法错误更难查。
